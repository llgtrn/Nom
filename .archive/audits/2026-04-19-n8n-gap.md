## 1. Pattern Summary

n8n implements a **defense-in-depth** strategy for safe execution of user-provided expressions and credential handling across four layers:

| Layer | n8n Implementation | Nom Equivalent |
|-------|-------------------|----------------|
| **L1 — Expression Sandbox** | Tournament AST parser → AST sanitizers (`ThisSanitizer`, `PrototypeSanitizer`, `DollarSignValidator`) → dual engine (in-process or `isolated-vm` V8 isolate) | Custom minimal AST evaluator in `nom-canvas-graph/src/sandbox.rs` with 7 sanitizers, no parser bridge |
| **L2 — Runtime Property Guards** | `__sanitize` runtime function, `SafeObject` Proxy, `SafeError` Proxy, blocked `constructor` regex | Static AST checks only (`PrototypeBlockSanitizer`, `ThisReplaceSanitizer`) |
| **L3 — Credential Encryption** | AES-256-CBC with per-instance key (`Cipher`); RBAC with scopes/roles; redaction/unredaction cycle | Plaintext `HashMap<String, Credential>` in `credential_store.rs` |
| **L4 — Node Execution Isolation** | Per-node `NodeExecutionContext` / `BaseExecuteContext`; `establishExecutionContext` with redaction policy; credential injection via `_getCredentials()` | `WorkflowRunner` logs node names only; no execution context abstraction; no credential injection |

## 2. Key Source Files (n8n)

### Expression Evaluation & Sandbox
- `packages/workflow/src/expression-sandboxing.ts` — AST sanitizers
  - `ThisSanitizer`: rewrites `this`, IIFE `.call()`, function `.bind()` to `EMPTY_CONTEXT` so user code cannot reach the Node.js global.
  - `PrototypeSanitizer`: blocks `__proto__`, `prototype`, `constructor` spread; validates variable declarations, catch clauses, class extensions against reserved names; wraps computed member expressions with `__sanitize()`.
  - `DollarSignValidator`: restricts `$` usage to function-call or property-access contexts.
- `packages/workflow/src/expression-evaluator-proxy.ts` — wraps `Tournament` with before/after hooks.
- `packages/workflow/src/expression.ts` — `Expression` class. Dual-engine architecture (`current` vs `vm`). `initializeGlobalContext()` builds an allowlist/denylist global object.
- `packages/@n8n/expression-runtime/src/bridge/isolated-vm-bridge.ts` — `IsolatedVmBridge`
  - Creates V8 isolate with `memoryLimit: 128` MB.
  - Compiles scripts with caching; enforces timeout via `script.runSync(..., { timeout })`.
  - Registers `ivm.Reference` callbacks (`__getValueAtPath`, `__getArrayElement`, `__callFunctionAtPath`) for cross-isolate data access.
  - Blocks native functions by checking `[native code]` in `fn.toString()`.
- `packages/@n8n/expression-runtime/src/evaluator/expression-evaluator.ts` — `ExpressionEvaluator`
  - Caches transformed code (`codeCache`) to avoid re-parsing AST.
  - Bridges to `IsolatedVmBridge.execute()`.

### Credential Store & Encryption
- `packages/core/src/encryption/cipher.ts` — `Cipher`
  - `encrypt(data)` / `decrypt(data)`: AES-256-CBC with random 8-byte salt, MD5-derived key/IV, CryptoJS-compatible header (`53616c7465645f5f`).
- `packages/core/src/credentials.ts` — `Credentials<T>`
  - `setData()` encrypts plaintext object → base64 string.
  - `getData()` decrypts → JSON parse.
  - `updateData()` decrypt-modify-re-encrypt cycle.
- `packages/cli/src/credentials/credentials.service.ts` — `CredentialsService`
  - `decrypt()` with redaction by default; `getOne()` scope-gated (`credential:read` vs `credential:update`).
  - `redactValues()` blanks password fields using `CREDENTIAL_BLANKING_VALUE` / `CREDENTIAL_EMPTY_VALUE`.
  - `unredact()` merges redacted frontend payload with stored decrypted data.
  - `DynamicCredentialsProxy` integration for external secrets.
- `packages/cli/src/credentials/dynamic-credentials-proxy.ts` — `DynamicCredentialsProxy`
  - `resolveIfNeeded()` / `storeIfNeeded()` for runtime credential resolution.
  - `storeOAuthTokenDataIfNeeded()` decrypts execution context to derive identity.

### Node Execution Isolation
- `packages/core/src/execution-engine/execution-context.ts` — `establishExecutionContext()`
  - Creates `runtimeData` with `version`, `establishedAt`, `source`, `redaction.policy`.
  - Inherits parent execution context for sub-workflows.
- `packages/core/src/execution-engine/node-execution-context/node-execution-context.ts` — `NodeExecutionContext`
  - `_getCredentials(type, itemIndex)`:
    1. Validates node declares the requested credential type.
    2. Checks `fullAccess` whitelist (`HTTP_REQUEST_NODE_TYPE`, etc.).
    3. Calls `credentialsHelper.getDecrypted()` with `expressionResolveValues`.
    4. Returns decrypted credential object to the node.
- `packages/core/src/execution-engine/node-execution-context/base-execute-context.ts` — `BaseExecuteContext`
  - `getCredentials()` wrapper, `executeWorkflow()` for sub-workflow isolation.

## 3. Gap Analysis

### 3.1 Expression Sandbox — AST & Execution

| Gap | n8n Capability | Nom Current State | Risk |
|-----|---------------|-------------------|------|
| **G1.1** No parser bridge | Tournament parses `{{ $json.email }}` into AST, then runs AST sanitizers. | `sandbox.rs` defines `Expr` enum but there is **no parser** from `.nomx` or `.nom` strings into `Expr`. The only way in is manual construction. | **HIGH** — sandbox cannot be used with user input. |
| **G1.2** No `__sanitize` runtime guard | `PrototypeSanitizer` wraps computed member expressions with `__sanitize(property)` which throws at runtime if the property is unsafe. | `PrototypeBlockSanitizer` only blocks literal variable names containing `__proto__`/`prototype`/`constructor`. It does **not** intercept computed dynamic property access at evaluation time. | **HIGH** — dynamic property access bypasses static checks. |
| **G1.3** No safe global wrappers | `Expression.initializeGlobalContext()` installs `SafeObject` (Proxy blocking `defineProperty`, `setPrototypeOf`, etc.) and `SafeError` (Proxy blocking `captureStackTrace`, `prepareStackTrace`). | `EvalContext` is a plain `HashMap<String, SandboxValue>`. No global object, no Proxy wrappers. | **MEDIUM** — if sandbox globals ever leak, V8 stack-trace escape is possible. |
| **G1.4** No constructor block | Regex `/\.\s*constructor/gm` rejects any expression containing `.constructor`. | `PrototypeBlockSanitizer` blocks literal `constructor` variable, but not member-access chains like `obj.constructor`. | **HIGH** — prototype-pollution / sandbox escape vector. |
| **G1.5** No VM isolation | `IsolatedVmBridge` runs expressions in a V8 isolate with 128 MB memory limit and configurable timeout (default 5 s). | `eval_expr` runs in-process with only a recursion depth limit (64). No memory cap, no timeout. | **CRITICAL** — untrusted code can DoS the Nom process. |
| **G1.6** No expression engine feature gate | `Expression.expressionEngine` can switch between `current` and `vm` via env var or API. | No pluggable evaluator interface. | **LOW** — limits future hardening without refactoring. |
| **G1.7** `NoSideEffectsSanitizer` is stub | n8n blocks `eval`, `Function`, `setTimeout`, etc. via global denylist. | `NoSideEffectsSanitizer::check()` returns `Ok(())` unconditionally with a TODO comment: *"implement before adding Expr::Assign/Import/Exec AST variants"*. | **HIGH** — once AST is extended, side effects will pass unchecked. |

### 3.2 Credential Store & Encryption

| Gap | n8n Capability | Nom Current State | Risk |
|-----|---------------|-------------------|------|
| **G2.1** No encryption at rest | `Cipher.encrypt()` → AES-256-CBC with per-instance key. | `CredentialStore` stores `Credential { kind, value }` as plaintext `String`. | **CRITICAL** — secrets are in memory and on disk in plaintext. |
| **G2.2** No instance-derived key | `Cipher` derives key from `InstanceSettings.encryptionKey` (randomly generated on first startup). | No key derivation, no `InstanceSettings` equivalent. | **HIGH** — even if encryption is added, keys must be managed. |
| **G2.3** No RBAC / scopes | `CredentialsService.getMany()` gates `includeData` on `credential:update` scope. `getSharing()` checks project roles. | `CredentialStore` has no user or scope concept. Any code with a `&CredentialStore` reference can read all secrets. | **HIGH** — no least-privilege access control. |
| **G2.4** No redaction / unredact cycle | `redact()` replaces passwords with sentinel values before sending to frontend; `unredact()` merges frontend payload back with stored secrets. | `Credential::fmt::Debug` redacts value, but there is no redaction for serialization/network transfer. | **MEDIUM** — secrets may leak over APIs or logs. |
| **G2.5** No dynamic credential resolution | `DynamicCredentialsProxy` resolves credentials at runtime from external secret providers. | Static HashMap only. | **MEDIUM** — prevents integration with vaults (HashiCorp, AWS SM, etc.). |
| **G2.6** No credential validation | `CredentialsService.validateOAuthCredentialUrls()`, `validateEntity()`. | No validation on credential shape or URL safety. | **MEDIUM** — malformed credentials can cause runtime errors or injection. |

### 3.3 Node Execution Isolation

| Gap | n8n Capability | Nom Current State | Risk |
|-----|---------------|-------------------|------|
| **G3.1** No execution context abstraction | `establishExecutionContext()` creates per-run `runtimeData` with redaction policy, source, timestamp. | `WorkflowRunner` is a bare DAG logger. No `ExecutionContext` struct. | **HIGH** — no audit trail, no policy enforcement per run. |
| **G3.2** No node-level context | `NodeExecutionContext` gives each node a typed API (`getCredentials`, `getNodeParameter`, `evaluateExpression`). | Nodes in `WorkflowGraph` are just `(id, name)` tuples. No parameter resolution, no credential injection. | **HIGH** — node cannot receive credentials or configuration securely. |
| **G3.3** No credential injection at runtime | `_getCredentials()` decrypts and injects credentials into the node execution context at the moment of execution. | `CredentialStore` is never referenced by `WorkflowRunner`, `CodeExecBackend`, or `ComposeOrchestrator`. | **CRITICAL** — the entire credential system is orphaned from the execution path. |
| **G3.4** No sub-workflow isolation | `BaseExecuteContext.executeWorkflow()` passes a new `RelatedExecution` with inherited context, but fresh `establishedAt`. | `ComposeOrchestrator.run_parallel()` calls `resolver.resolve()` sequentially with no context propagation. | **MEDIUM** — nested compositions share ambient state unsafely. |
| **G3.5** No abort signal / cancellation | `BaseExecuteContext` exposes `abortSignal` and `onExecutionCancellation()`. | `WorkflowRunner` has no cancellation mechanism. | **MEDIUM** — long-running sandbox evaluations cannot be interrupted. |

### 3.4 Safe JS/TS Execution in Workflows

| Gap | n8n Capability | Nom Current State | Risk |
|-----|---------------|-------------------|------|
| **G3.6** No HTML / webhook sandbox | `getWebhookSandboxCSP()` returns strict CSP sandbox directive for webhook responses. | No webhook server; no HTML sandboxing. | **LOW** — not yet in scope for Nom. |
| **G3.7** No SSRF guards in request helpers | `request-helpers/axios-config.ts` and `ssrf-integration.test.ts` implement SSRF filtering. | `nom-compose/src/backends/code_exec.rs` does not make HTTP requests, but future vendor integrations will. | **MEDIUM** — referenced in `Spider` adoption plan but not yet implemented. |

### Total Effort (realistic)
- **Minimum viable (P0 only):** 2.5–3 weeks
- **Full parity with n8n patterns:** 4–5 weeks

### Recommended Shortest Path
1. **Week 1:** Parser bridge + wire sandbox + `NoSideEffectsSanitizer` + constructor block.
2. **Week 2:** Credential encryption + instance key + node execution context + credential injection.
3. **Week 3:** VM isolation stub (feature-gated) + RBAC/redaction + dynamic proxy trait.

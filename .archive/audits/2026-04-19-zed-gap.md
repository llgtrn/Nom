## 1. Pattern Summary (What Zed Does That Nom Should Replicate)

Zed's GPUI is a **full retained-mode GPU UI framework** with the following end-to-end flow every frame:

```
Platform event loop
  └─ on_request_frame callback
       ├─ if dirty → Window::draw(cx)
       │    ├─ invalidate_entities()          // mark changed views
       │    ├─ draw_roots(cx)
       │    │    ├─ prepaint phase            // layout + hitboxes + dispatch tree
       │    │    │    └─ root_element.prepaint_as_root(...)
       │    │    ├─ paint phase               // emit primitives to Scene
       │    │    │    └─ root_element.paint(...)
       │    │    └─ paint deferred (tooltips, drag, prompts)
       │    ├─ swap rendered_frame ↔ next_frame
       │    ├─ focus change notifications
       │    └─ return ArenaClearNeeded
       ├─ Window::present()
       │    └─ platform_window.draw(&rendered_frame.scene)
       │         └─ WgpuRenderer::draw(scene)  // GPU submit + present
       └─ complete_frame()
```

Key architectural commitments in Zed:
1. **Dirty-tracking invalidator** — only redraws when views or input change.
2. **Double-buffered frames** — `rendered_frame` (last presented) vs `next_frame` (building).
3. **Three-phase element lifecycle** — `request_layout` → `prepaint` → `paint` with typed state.
4. **Scene graph with ordered batching** — `BoundsTree` gives each primitive a `DrawOrder`; `BatchIterator` interleaves batches across primitive types by z-order.
5. **Atlas-driven sprite rendering** — `PlatformAtlas` / `WgpuAtlas` manages glyph/image texture tiles.
6. **Per-platform `PlatformWindow` trait** — abstracts surface creation, input, resize, present.
7. **Text system integration** — `WindowTextSystem` + glyph cache + subpixel variants.

## 3. Gap Analysis — Nom vs Zed

### 3.1 Window State & Event Loop (`window.rs`)

| Capability | Zed | Nom | Gap Severity |
|------------|-----|-----|--------------|
| **Dirty invalidation** | `WindowInvalidator` tracks dirty views, draw phases (`None`/`Prepaint`/`Paint`/`Focus`), and prevents redundant redraws. | `frame_pending: bool` only. No view-level dirty tracking. | **HIGH** |
| **Double-buffered frames** | `rendered_frame: Frame` and `next_frame: Frame` swapped every draw; retains hitboxes, dispatch tree, scene, element states. | No frame buffering. Scene is created fresh in `RedrawRequested`. | **HIGH** |
| **Layout engine** | `TaffyLayoutEngine` computes flexbox-style layouts; `request_layout` returns real `LayoutId`. | `LayoutRegistry` hands out monotonically increasing `u64` IDs with **no actual layout computation**. `WindowContext::layout()` returns `Bounds::default()`. | **CRITICAL** |
| **Focus system** | `FocusId`, `FocusHandle`, tab stops, focus path, focus listeners, `DispatchTree` for keyboard event routing. | No focus types, no dispatch tree, no tab stops. | **HIGH** |
| **Hit testing** | `Hitbox`, `HitboxId`, `HitboxBehavior` (Normal / BlockMouse / BlockMouseExceptScroll), `mouse_hit_test` computed per frame from `next_frame.hit_test()`. | `cursor_position: Vec2` stored, but **no hitboxes, no hit testing infrastructure**. | **HIGH** |
| **Text system** | `WindowTextSystem` integrated per window; text style stacks; glyph layout caching. | No text system in window. `text_layout.rs` exists but is not wired to window. | **MEDIUM** |
| **Platform abstraction** | `PlatformWindow` trait with 30+ methods: `draw`, `resize`, `on_input`, `on_request_frame`, `sprite_atlas`, `modifiers`, etc. | `Platform` trait has 4 methods (`create_surface_descriptor`, `adapter_options`, `present_mode`, `device_features`). No `PlatformWindow` abstraction — winit is hard-coded inside `run_native_application`. | **HIGH** |
| **Event loop ownership** | `App` owns the event loop; platform window calls back into `Window` via `on_request_frame`. | `run_native_application` owns the winit event loop directly in `window.rs`, mixing application and window logic. | **MEDIUM** |
| **Resize handling** | `bounds_changed()` updates `viewport_size`, `scale_factor`, `display_id`, then `refresh()`. | Updates `content_size` but does **not reconfigure wgpu surface** on resize (only stores value). | **HIGH** |
| **Thermal / VRR** | `InputRateTracker` sustains presentation during high-rate input; thermal throttling skips frames. | No input rate tracking or thermal awareness. | **LOW** |

**Cited Zed source:**
- `window.rs:104-188` — `WindowInvalidator`
- `window.rs:747-901` — `Frame` struct with scene, hitboxes, dispatch tree, deferred_draws
- `window.rs:1116-1474` — `Window::new()` initializing all subsystems
- `window.rs:2246-2319` — `Window::draw()` full frame lifecycle
- `window.rs:2344-2348` — `Window::present()` -> `platform_window.draw(scene)`

### 3.2 Scene Graph & Batching (`scene.rs`)

| Capability | Zed | Nom | Gap Severity |
|------------|-----|-----|--------------|
| **Primitive ordering** | Each primitive gets a `DrawOrder` from `BoundsTree<ScaledPixels>`; `insert_primitive()` assigns `order`. | No `DrawOrder`. Primitives are pushed to typed `Vec`s in insertion order. | **HIGH** |
| **Layer stack** | `push_layer(bounds)` / `pop_layer()` with `layer_stack: Vec<DrawOrder>` for clipping / z-grouping. | No layer stack. `FrostedRect` exists but no GPU blur pass. | **HIGH** |
| **Ordered batching** | `BatchIterator` interleaves `PrimitiveBatch` variants (Shadows -> Quads -> Paths -> Sprites -> Underlines) sorted by `(order, kind)`. | `sort_and_batch()` only sorts sprites by `texture_id`. **No interleaving across primitive types** — all quads draw before all paths regardless of z-order. | **CRITICAL** |
| **Paint operations log** | `paint_operations: Vec<PaintOperation>` records every `StartLayer`, `EndLayer`, and `Primitive` for replay and diffing. | No `paint_operations`. Scene is just buckets. | **MEDIUM** |
| **Path geometry** | `Path<Pixels>` supports `move_to`, `line_to`, `curve_to`, `push_triangle`; scaled to `ScaledPixels` for GPU. | `Path` struct exists but no builder API; vertices are stored directly. | **MEDIUM** |
| **Surface primitives** | `PaintSurface` for platform video frames (macOS `CVPixelBuffer`). | No surface primitive. | **LOW** |

**Cited Zed source:**
- `scene.rs:25-178` — `Scene` with `paint_operations`, `primitive_bounds`, `layer_stack`, `batches()`
- `scene.rs:255-452` — `BatchIterator` interleaving by `(order, kind)`
- `scene.rs:733-858` — `Path<Pixels>` builder API

### 3.3 Element Lifecycle (`element.rs`)

| Capability | Zed | Nom | Gap Severity |
|------------|-----|-----|--------------|
| **Two-state element trait** | `Element::RequestLayoutState` + `Element::PrepaintState` — type-safe state carried across phases. | Single `Element::State` associated type. No prepaint state separation. | **MEDIUM** |
| **Arena allocation** | `AnyElement` = `ArenaBox<dyn ElementObject>`; allocated in per-App arena during draw, cleared after `ArenaClearNeeded::clear()`. | `AnyElement` is a manual trait (`request_layout_dyn`, `prepaint_dyn`, `paint_dyn`). No arena, no batch deallocation. | **MEDIUM** |
| **Drawable state machine** | `Drawable<E>` tracks `ElementDrawPhase` (Start -> RequestLayout -> LayoutComputed -> Prepaint -> Painted) preventing phase skew. | No `Drawable` wrapper. Elements are called directly; no phase enforcement. | **MEDIUM** |
| **Global element ID** | `GlobalElementId(Arc<[ElementId]>)` tracks elements across frames for state persistence. | `GlobalElementId` exists but is a `Vec<ElementId>` wrapper; no cross-frame state persistence. | **MEDIUM** |
| **Source location** | `Element::source_location()` for inspector / debug. | No source location tracking. | **LOW** |
| **IntoElement / RenderOnce** | Derive macros and `Component<C>` wrapper for fluent UI DSL. | `Div` is hand-rolled; no derive macros, no `RenderOnce` pattern. | **MEDIUM** |

**Cited Zed source:**
- `element.rs:51-110` — `Element` trait with `RequestLayoutState`, `PrepaintState`
- `element.rs:322-555` — `Drawable<E>` with `ElementDrawPhase` state machine

### 3.4 Renderer -> GPU Submission (`renderer.rs` / `wgpu_renderer.rs`)

| Capability | Zed | Nom | Gap Severity |
|------------|-----|-----|--------------|
| **Pipeline completeness** | 8 live pipelines: quads, shadows, path_rasterization, paths, underlines, mono_sprites, subpixel_sprites, poly_sprites, surfaces. | 1 live pipeline: `quad_pipeline`. 6 pipelines are **stats-only stubs** (shadows, paths, mono sprites, poly sprites, underlines, frosted). 2 slots reserved. | **CRITICAL** |
| **Atlas texture management** | `WgpuAtlas` with texture tiles, `AtlasTextureId`, `get_texture_info()`, sampler, bind groups per texture. | `AtlasTile` type exists in `types.rs`, but **no atlas allocator or GPU textures** in `Renderer`. `texture_atlas.rs` exists but is not wired to renderer. | **CRITICAL** |
| **Instance buffer overflow** | `grow_instance_buffer()` doubles capacity; loop retries draw on overflow. | `GpuResources::ensure_capacity()` exists for quad buffer but **only for quads**; no overflow retry loop. | **MEDIUM** |
| **Path rendering** | Two-pass path rasterization to intermediate texture (`path_intermediate_texture`, `path_msaa_texture`), then composited to main pass. | `draw_paths()` increments `paths_drawn` stat and returns. **No GPU path rendering**. | **HIGH** |
| **Shadow rendering** | Dedicated `shadows` pipeline with blur kernel (GPU-side). | `draw_shadows()` increments `shadows_drawn` stat. **No GPU shadow rendering**. | **HIGH** |
| **Sprite rendering** | `draw_monochrome_sprites()`, `draw_subpixel_sprites()`, `draw_polychrome_sprites()` with texture binding per atlas tile. | `draw_monochrome_sprites()` / `draw_polychrome_sprites()` increment stats. **No GPU sprite rendering**. | **HIGH** |
| **Underline rendering** | Dedicated `underlines` pipeline. | `draw_underlines()` increments stats. **No GPU underline rendering**. | **MEDIUM** |
| **Frosted glass** | Not directly in Zed (Zed uses backdrop blur via platform / shader). Nom has a software approximation. | `draw_frosted_rects()` decomposes to 2 quads per rect with alpha blending. **No Gaussian blur pre-pass**. | **MEDIUM** |
| **Global uniforms** | `GlobalParams` (viewport size, premultiplied alpha), `GammaParams`, `PathGlobals` uploaded to GPU per frame. | `GlobalUniforms` struct defined but **never uploaded** — `quad_pipeline` shader receives no uniform buffer binding in `end_frame_render`. | **HIGH** |
| **Surface acquire & error handling** | Handles `Suboptimal`, `Lost`, `Outdated`, `Timeout`, `Occluded`, `Validation` with reconfigure and error counting. | Handles `Lost`, `Outdated`, `Timeout`, `OutOfMemory` but skips frame instead of reconfiguring surface. No `failed_frame_count` or validation error tracking. | **MEDIUM** |
| **Presenter separation** | `PlatformWindow::draw(scene)` delegates to `WgpuRenderer::draw(scene)` which acquires surface, encodes, submits, presents. | `window.rs` directly calls `renderer.begin_frame()`, `renderer.draw(scene)`, `renderer.end_frame_render(surface, device, queue)` — no `PlatformWindow` abstraction. | **MEDIUM** |

**Cited Zed source:**
- `wgpu_renderer.rs:83-94` — `WgpuPipelines` (8 pipelines)
- `wgpu_renderer.rs:1073-1316` — `WgpuRenderer::draw(scene)` with batch loop, overflow retry, surface acquire
- `wgpu_renderer.rs:1318-1458` — `draw_quads()`, `draw_shadows()`, `draw_instances()` with buffer write + bind group
- `wgpu_renderer.rs:1429-1458` — `draw_instances()` writing to instance buffer, creating bind groups, issuing `pass.draw(0..4, ...)`

## 4. Adoption Effort Estimate

| Gap Area | Effort | Blocker for Demo? | Notes |
|----------|--------|-------------------|-------|
| **Wire `end_frame_render` into event loop** | 1–2 days | **YES (CB1)** | Already partially done in `window.rs:339-369`, but needs surface reconfiguration on resize. |
| **Connect scene -> GPU quad upload** | 2–3 days | **YES (CB2)** | `Renderer::draw()` already converts quads to `QuadInstance`, but `end_frame_render` needs to bind uniform buffer and set correct vertex buffer stride. |
| **Add 6 missing GPU pipelines** | 2–3 weeks | No | Shadows, paths, sprites, underlines each need WGSL shaders + pipeline descriptors + instance buffer layouts. |
| **Integrate atlas allocator -> renderer** | 1 week | No | `texture_atlas.rs` has `AtlasAllocator`; needs `WgpuAtlas`-style texture creation and bind-group binding in sprite draw calls. |
| **Implement real layout engine (Taffy)** | 1–2 weeks | No | `nom-gpui/src/layout.rs` may already exist; needs wiring into `WindowContext::request_layout()`. |
| **Add `PlatformWindow` abstraction** | 3–5 days | No | Split winit-specific code out of `window.rs` into a `PlatformWindow` impl. |
| **Double-buffered frames + invalidator** | 1 week | No | Needed for efficient redraws; without it every frame rebuilds everything. |
| **Dispatch tree + hit testing** | 1–2 weeks | No | Core for interactive UI; can be deferred if initial demo is static. |
| **Text system integration** | 1–2 weeks | No | `text_layout.rs` exists but needs glyph atlas and layout caching. |
| **Scene batching by DrawOrder** | 3–5 days | No | Rewrite `sort_and_batch()` to emit `PrimitiveBatch` interleaving like Zed's `BatchIterator`. |

**Total to match Zed's full GPUI:** ~8–10 weeks of focused renderer work.  
**Total to render a colored quad end-to-end:** Already mostly there; fix uniform buffer binding + surface resize reconfiguration (~2–3 days).

1. Core compile pipeline
   - lexer
   - parser
   - resolver
   - verifier
   - planner
   - LLVM/codegen

2. Dictionary and concept system
   - `nom-dict`
   - `nom-concept`
   - staged parsing / strictness / synonym flows

3. Application/report surfaces
   - `nom-app`
   - `nom-cli`
   - build/status/report/dream commands

4. Newer frontier work
   - `nom-grammar`
   - `nom-intent`
   - `nom-lsp`

5. Broader platform/media/runtime layer
   - locale
   - media
   - flow
   - bench
   - runtime
   - security
   - graph

The repo is therefore not broken everywhere. It is better described as:

- a still-coherent compiler core
- plus several newer subsystems landing faster than the cleanup and documentation loop
